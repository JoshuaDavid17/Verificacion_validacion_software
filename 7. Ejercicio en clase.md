# 7. Ejercicio en clase: Crea tres historias de usuario para una aplicación bancaria móvil.

### Historia de Usuario 1: Enviar comprobante de pago

Como usuario, <br>
quiero enviar el comprobante de una transferencia,<br>
para compartir la evidencia del pago con otra persona.

### Criterios de aceptación

- El sistema debe permitir enviar el comprobante por WhatsApp, correo o descargarlo en PDF.
- El comprobante debe incluir fecha, monto, cuenta origen, cuenta destino y número de transacción.
- El comprobante debe generarse automáticamente después de completarse la transferencia.
- El usuario debe poder reenviar el comprobante desde el historial de transacciones.
- El sistema debe mostrar una confirmación de que el comprobante fue enviado correctamente.
***
### Historia de Usuario 2: Consulta de movimientos

Como cliente del banco,<br>
quiero ver los movimientos recientes de mis cuentas,<br>
para llevar un control detallado de mis transacciones.

### Criterios de aceptación

- El sistema debe mostrar al menos los últimos 30 movimientos de cada cuenta.
- Cada movimiento debe incluir fecha, tipo, monto y saldo actual.
- El usuario debe poder filtrar movimientos por fecha, tipo o monto.
- La lista de movimientos debe actualizarse automáticamente al detectar nuevas transacciones.
- El sistema debe permitir exportar los movimientos en formato PDF o CSV.
---

### Historia de Usuario 3: Transferencias entre cuentas

Como usuario de la app bancaria,<br>
quiero realizar transferencias entre mis cuentas,<br>
para mover dinero rápidamente sin ir al banco.

### Criterios de aceptación

- El sistema debe permitir seleccionar la cuenta origen y destino.
- Debe mostrar un mensaje de confirmación con el monto y detalle antes de enviar la transferencia.
- La app debe validar que la cuenta origen tenga saldo suficiente antes de permitir la operación.
- La transferencia debe reflejarse inmediatamente en el saldo de ambas cuentas.
- El usuario debe recibir una notificación push confirmando la transferencia realizada.
