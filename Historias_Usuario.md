# Ejemplo 1: Sistema de compras en línea

### Historia de usuario
Como cliente,  
quiero agregar productos a mi carrito de compras,  
para poder revisarlos antes de realizar el pago.

### Criterios de aceptación
- Debe permitir agregar y eliminar productos del carrito.  
- Debe mostrar el subtotal, impuestos y total.  
- El carrito debe mantenerse incluso si el usuario recarga la página o inicia sesión más tarde.

### Tipo de usuario
Cliente

### Prioridad
Alta

### Valor:
Permite revisar y organizar sus compras antes de pagar, mejorando la experiencia de compra.


# Ejemplo 2: Sistema de turnos médicos

### Historia de usuario
Como paciente,  
quiero reservar una cita médica en línea,  
para evitar filas y administrar mi tiempo.

### Criterios de aceptación
- Debe mostrar fechas y horarios disponibles según el médico.  
- Debe enviar una confirmación por correo al reservar.  
- Permitir reprogramar o cancelar la cita.

### Tipo de usuario:
Paciente

### Prioridad:
Alta (Es una funcionalidad principal del sistema)

### Valor:
Permite disminuir filas, mejorar la experiencia del paciente y optimizar la gestión de citas médicas.


# Ejemplo 3: Sistema de control de medicamentos próximos a caducar

### Historia de usuario
Como encargado de farmacia,  
quiero recibir alertas de medicamentos próximos a vencer,  
para retirarlos a tiempo y evitar pérdidas o riesgos.

### Criterios de aceptación
- Calcular automáticamente qué productos vencen en los próximos 30, 60 y 90 días.  
- Enviar notificaciones diarias o semanales.  
- Permitir generar un reporte PDF con la lista de productos próximos a caducar.
  
### Tipo de usuario:
Encargado de farmacia

### Prioridad:
Alta -> Evitar pérdidas económicas y riesgos para los pacientes es crítico.

### Valor:
Reduce pérdidas por vencimiento, mejora el control de inventario y garantiza la seguridad del paciente.
  

# Ejemplo 4: Biblioteca digital

### Historia de usuario
Como lector,  
quiero buscar libros por título o autor,  
para encontrar rápidamente el material que necesito.

### Criterios de aceptación
- Debe tener un buscador con filtros avanzados (autor, año, género).  
- Debe mostrar disponibilidad (prestado o disponible).  
- Permitir reservar un libro si está ocupado.
  
### Tipo de usuario:
Lector

### Prioridad:
Media-Alta -> La búsqueda es esencial para la usabilidad de la biblioteca digital.

### Valor:
Facilita el acceso rápido a los libros, mejora la experiencia del usuario y aumenta el uso de la plataforma.

# Ejemplo 5: Sistema de seguimiento para mascotas rescatadas

### Historia de usuario
Como voluntario de un refugio,  
quiero registrar la evolución de salud de cada mascota rescatada,  
para asegurar que reciban la atención adecuada.

### Criterios de aceptación
- Registrar peso, vacunas, comportamiento y tratamientos.  
- Adjuntar fotos del antes y después.  
- Mostrar alertas cuando una mascota necesita revisión médica.

### Tipo de usuario:
Voluntario de refugio animal

### Prioridad:
Alta -> Es información crítica para garantizar el bienestar de las mascotas rescatadas.

### Valor:
Permite llevar un control adecuado de la salud de cada mascota, optimiza los cuidados y facilita la toma de decisiones en el refugio.
